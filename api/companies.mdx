## **GET** /api/companies

List all companies. Results are scoped by role: `PAYPAC` can see all companies and filter by status; `ORGANIZER` sees only their own company; other roles see only approved companies.

**Required role:** Any authenticated user

### Request headers

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

### Query parameters

<ParamField query="search" type="string">
  Text search against company name.
</ParamField>

<ParamField query="country_id" type="number">
  Filter by country ID.
</ParamField>

<ParamField query="state_id" type="number">
  Filter by state/department ID.
</ParamField>

<ParamField query="city_id" type="number">
  Filter by city ID.
</ParamField>

<ParamField query="status" type="number">
  Filter by status. `0` = pending, `1` = approved, `2` = suspended. Only usable by `PAYPAC`.
</ParamField>

### Response `200`

<ResponseField name="message" type="string">
  `"Empresas obtenidas exitosamente"`
</ResponseField>

<ResponseField name="data" type="object[]">
  Array of company objects.

  <Expandable title="company properties">
    <ResponseField name="id" type="number">Company ID.</ResponseField>
    <ResponseField name="company_name" type="string">Legal company name. Unique.</ResponseField>
    <ResponseField name="company_description" type="string">Short description.</ResponseField>
    <ResponseField name="company_logo" type="string">URL to the company logo image.</ResponseField>
    <ResponseField name="company_cover" type="string">URL to the company cover image.</ResponseField>
    <ResponseField name="company_phone_number" type="string">Contact phone number.</ResponseField>
    <ResponseField name="company_email" type="string">Contact email. Unique.</ResponseField>
    <ResponseField name="type_identification" type="string">Identification document type (e.g. `NIT`).</ResponseField>
    <ResponseField name="num_identification" type="string">Identification document number.</ResponseField>
    <ResponseField name="website" type="string">Company website URL.</ResponseField>
    <ResponseField name="address" type="string">Physical address.</ResponseField>
    <ResponseField name="company_presentation" type="string">Extended presentation text.</ResponseField>
    <ResponseField name="rating" type="number">Average rating (0â€“5).</ResponseField>
    <ResponseField name="status" type="number">`0` = pending, `1` = approved, `2` = suspended.</ResponseField>
    <ResponseField name="country_id" type="number">Country FK.</ResponseField>
    <ResponseField name="state_id" type="number">State FK.</ResponseField>
    <ResponseField name="city_id" type="number">City FK.</ResponseField>
    <ResponseField name="approved_at" type="string">ISO 8601 timestamp of approval, or `null`.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Query validation error. |
| `401`  | Missing or invalid Bearer token. |

<CodeGroup>

```bash cURL
curl --request GET \
  --url 'https://api.paypac.com.co/api/companies?status=1&country_id=1' \
  --header 'Authorization: Bearer <firebase-id-token>'
```

</CodeGroup>

## **POST** /api/companies

Create a new company. When called by an `ORGANIZER`, the company is automatically linked to their user account. When called by `PAYPAC`, the company starts in pending status.

**Required role:** `PAYPAC` or `ORGANIZER`

### Request headers

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

### Request body

<ParamField body="company_name" type="string" required>
  Legal company name. Must be unique across all companies.
</ParamField>

<ParamField body="company_description" type="string">
  Short description of the company.
</ParamField>

<ParamField body="company_logo" type="string">
  URL to the company logo image.
</ParamField>

<ParamField body="company_cover" type="string">
  URL to the company cover image.
</ParamField>

<ParamField body="company_phone_number" type="string">
  Contact phone number.
</ParamField>

<ParamField body="company_email" type="string">
  Contact email address. Must be unique.
</ParamField>

<ParamField body="type_identification" type="string">
  Document type used for the company identification (e.g. `NIT`).
</ParamField>

<ParamField body="num_identification" type="string">
  Identification document number.
</ParamField>

<ParamField body="website" type="string">
  Company website URL.
</ParamField>

<ParamField body="address" type="string">
  Physical address.
</ParamField>

<ParamField body="company_presentation" type="string">
  Extended presentation text shown on the public profile.
</ParamField>

<ParamField body="country_id" type="number">
  Country FK. References `Countries.id`.
</ParamField>

<ParamField body="state_id" type="number">
  State/department FK. References `States.id`.
</ParamField>

<ParamField body="city_id" type="number">
  City FK. References `Cities.id`.
</ParamField>

### Response `201`

<ResponseField name="message" type="string">
  `"Empresa creada exitosamente"`
</ResponseField>

<ResponseField name="data" type="object">
  The newly created company object.
</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Validation error or company name/email already exists. |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Authenticated user does not have `PAYPAC` or `ORGANIZER` role. |

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.paypac.com.co/api/companies \
  --header 'Authorization: Bearer <firebase-id-token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "company_name": "Eventos BogotĂˇ SAS",
    "company_email": "info@eventosbogota.com",
    "company_phone_number": "3001234567",
    "type_identification": "NIT",
    "num_identification": "900123456-1",
    "country_id": 1,
    "state_id": 3,
    "city_id": 11
  }'
```

</CodeGroup>

**Required role:** `PAYPAC` or `ORGANIZER`

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

### Path parameters

<ParamField path="id" type="number" required>
  Company ID.
</ParamField>

### Request body

All fields are optional. Only provided fields are updated.

<ParamField body="company_name" type="string">
  Legal company name. Must remain unique.
</ParamField>

<ParamField body="company_description" type="string">
  Short description.
</ParamField>

<ParamField body="company_logo" type="string">
  URL to the logo image.
</ParamField>

<ParamField body="company_cover" type="string">
  URL to the cover image.
</ParamField>

<ParamField body="company_phone_number" type="string">
  Contact phone number.
</ParamField>

<ParamField body="company_email" type="string">
  Contact email address.
</ParamField>

<ParamField body="type_identification" type="string">
  Identification document type.
</ParamField>

<ParamField body="num_identification" type="string">
  Identification document number.
</ParamField>

<ParamField body="website" type="string">
  Company website URL.
</ParamField>

<ParamField body="address" type="string">
  Physical address.
</ParamField>

<ParamField body="company_presentation" type="string">
  Extended presentation text.
</ParamField>

<ParamField body="country_id" type="number">
  Country FK.
</ParamField>

<ParamField body="state_id" type="number">
  State FK.
</ParamField>

<ParamField body="city_id" type="number">
  City FK.
</ParamField>

### Response `200`

<ResponseField name="message" type="string">
  `"Empresa actualizada exitosamente"`
</ResponseField>

<ResponseField name="data" type="object">
  The updated company object.
</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Validation error, or `ORGANIZER` attempted to edit an already-approved company. |

| `404`  | Company not found. |

  }'
