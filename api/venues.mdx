---
title: "Venues"
description: "Manage numbered venues with zones, rows, seats, and per-event seat status. Use these endpoints to build the seating chart for numbered-seat events."
---

<Info>
  Venues are used exclusively with events that have `type_venue: NUMERADO`. The data model follows a strict hierarchy:

  **Venue → Zone → Row → Seat → SeatStatus (per event)**

  A venue is a reusable physical space (e.g., a theatre). Zones subdivide the venue into sections. Each zone contains rows, each row contains seats, and each seat has a status record per event (`AVAILABLE`, `HELD`, `SOLD`, or `BLOCKED`).
</Info>

All endpoints require a valid Firebase ID token: `Authorization: Bearer <token>`.

---

## Venues

### **GET** /api/venues

Lists all venues with optional filtering by name, venue type, and seating format.

**Required role:** `PAYPAC`, `ORGANIZER`, `STAFF`, `STAFF_PROMOTER`, or `PROMOTER`

#### Request headers

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

#### Query parameters

<ParamField query="search" type="string">
  Search term matched against the venue name.
</ParamField>

<ParamField query="type_place" type="string">
  Filter by venue type. One of: `CLUB`, `DISCOTECA`, `TEATRO`, `CINE`, `ESTADIO`.
</ParamField>

<ParamField query="place_type" type="string">
  Filter by seating format. `NUMERADO` or `SIN_NUMERAR`.
</ParamField>

#### Response `200`

<ResponseField name="id" type="number">Venue ID.</ResponseField>
<ResponseField name="name_place" type="string">Venue name.</ResponseField>
<ResponseField name="type_place" type="string">Venue type (`CLUB`, `DISCOTECA`, `TEATRO`, `CINE`, `ESTADIO`).</ResponseField>
<ResponseField name="place_type" type="string">Seating format (`NUMERADO` or `SIN_NUMERAR`).</ResponseField>
<ResponseField name="capacity" type="number">Total seat count.</ResponseField>
<ResponseField name="map_place" type="object">Interactive map JSON, or `null` if not yet defined.</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>

#### Error responses

| Status | Description |
|--------|-------------|
| `401`  | Missing or invalid Bearer token. |
| `403`  | Insufficient role. |

<CodeGroup>

```bash cURL
curl --request GET \
  --url 'https://api.paypac.com.co/api/venues?type_place=TEATRO&search=Nacional' \
  --header 'Authorization: Bearer <firebase-id-token>'
```

</CodeGroup>

## Seat Status

Per-event seat status tracks the real-time state of each seat for a specific event. This is separate from the physical seat status and must be initialized before an event goes on sale.

| Status | Meaning |
|--------|---------|
| `AVAILABLE` | Seat is open for purchase. |
| `HELD` | Seat is in a customer's cart. Expires after 10 minutes if not purchased. |
| `SOLD` | Seat has been purchased. |
| `BLOCKED` | Seat is reserved for courtesy, press, or production. |

---

### **POST** /api/venues/seat-status/initialize

Initializes `EventSeatStatus` records for every `ACTIVE` seat in a venue for a given event. All seats start as `AVAILABLE`. Call this when approving a numbered-seat event.

**Required role:** `PAYPAC` or `ORGANIZER`

#### Request body

<ParamField body="place_id" type="number" required>
  ID of the venue whose seats to initialize.
</ParamField>

<ParamField body="event_id" type="number" required>
  ID of the event to create seat status records for.
</ParamField>

#### Response `201`

<ResponseField name="initialized" type="number">Number of seat status records created.</ResponseField>
<ResponseField name="event_id" type="number">Event ID.</ResponseField>
<ResponseField name="place_id" type="number">Venue ID.</ResponseField>

<CodeGroup>

### **POST** /api/venues/seat-status/hold

Moves a seat into `HELD` status for the current user's cart. The hold expires automatically after 10 minutes (`held_until` timestamp). If the seat is already `HELD` or `SOLD`, the request fails.

**Required role:** Any authenticated user (including `CUSTOMER`)

#### Request body

<ParamField body="seat_id" type="number" required>
  ID of the seat to hold.
</ParamField>

<ParamField body="event_id" type="number" required>
  ID of the event.
</ParamField>

#### Response `200`

<ResponseField name="seat_id" type="number">Seat ID.</ResponseField>
<ResponseField name="event_id" type="number">Event ID.</ResponseField>
<ResponseField name="status" type="string">`HELD`.</ResponseField>
<ResponseField name="held_until" type="string">ISO 8601 timestamp when the hold expires.</ResponseField>

#### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Seat is already `HELD` or `SOLD`. |
| `401`  | Missing or invalid Bearer token. |

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.paypac.com.co/api/venues/seat-status/hold \
  --header 'Authorization: Bearer <firebase-id-token>' \
  --header 'Content-Type: application/json' \
  --data '{"seat_id": 150, "event_id": 101}'
```

</CodeGroup>

### **GET** /api/venues/seat-status/:event_id/map

Returns a flat map of `{ seat_id: status }` for every seat in the event's venue. Used by the interactive seating chart to render seat availability in real time.

**Required role:** Any authenticated user (including `CUSTOMER`)

#### Path parameters

<ParamField path="event_id" type="number" required>
  Numeric event ID.
</ParamField>

#### Response `200`

<ResponseField name="event_id" type="number">Event ID.</ResponseField>
<ResponseField name="map" type="object">
  A flat key-value map where each key is a `seat_id` (as a string) and the value is the current `SeatEventStatus`: `AVAILABLE`, `HELD`, `SOLD`, or `BLOCKED`.

  Example: `{ "150": "AVAILABLE", "151": "HELD", "152": "SOLD" }`
</ResponseField>

<CodeGroup>

```bash cURL

  --url https://api.paypac.com.co/api/venues/seat-status/101/map \
  --header 'Authorization: Bearer <firebase-id-token>'
```
