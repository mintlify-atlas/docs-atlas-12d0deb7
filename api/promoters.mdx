## Promoter codes

Promoter codes are unique alphanumeric strings assigned to each promoter. When a customer applies a code at checkout, the promoter earns a commission based on the applicable reward rule for that event.

---

## Event reward rules

Reward rules define how much commission a promoter earns when their code is applied to a ticket purchase for a specific event. Each event can have one or more rules, optionally scoped to a locality.

### `EventRewardPromoters` enum

| Value | Description |
|-------|-------------|
| `NONE` | No reward. Used as a default placeholder. |
| `PERCENTAGE` | Commission is a percentage of the ticket sale amount. Configured via `reward_percentage`. |
| `FIXED_AMOUNT` | A flat monetary amount per qualifying sale. Configured via `reward_amount`. |
| `GUEST_LIST` | Reward takes the form of complimentary guest-list entries. |
| `TICKET_REWARD` | Reward takes the form of free tickets. |
| `CASH_REWARD` | Cash payout reward. |
| `CONSUMPTION_REWARD` | Reward in the form of event consumption credit. |

## **POST** /api/events/:eventId/reward-rules

Create a new reward rule for an event.

**Required role:** `ORGANIZER` (must own the event) or `PAYPAC`

### Request headers

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

### Path parameters

<ParamField path="eventId" type="number" required>
  Event ID.
</ParamField>

### Request body

<ParamField body="reward_type" type="string" required>
  Reward type. One of: `NONE`, `PERCENTAGE`, `FIXED_AMOUNT`, `GUEST_LIST`, `TICKET_REWARD`, `CASH_REWARD`, `CONSUMPTION_REWARD`.
</ParamField>

<ParamField body="reward_percentage" type="number">
  Percentage value (0–100). Required when `reward_type` is `PERCENTAGE`.
</ParamField>

<ParamField body="reward_amount" type="number">
  Fixed monetary amount in the event currency (integer, in minor units). Required when `reward_type` is `FIXED_AMOUNT` or `CASH_REWARD`.
</ParamField>

<ParamField body="min_qty_tickets" type="number">
  Minimum number of tickets in the order for this rule to apply.
</ParamField>

<ParamField body="min_amount_tickets" type="number">
  Minimum total order value (in minor units) for this rule to apply.
</ParamField>

<ParamField body="locality_id" type="number">
  Scope the rule to a specific event locality. If omitted, the rule applies to all localities.
</ParamField>

### Response `201`

<ResponseField name="message" type="string">
  `"Regla de recompensa creada exitosamente"`
</ResponseField>

<ResponseField name="rule" type="object">
  The created reward rule.

  <Expandable title="rule properties">
    <ResponseField name="id" type="number">Rule ID.</ResponseField>
    <ResponseField name="event_id" type="number">Associated event ID.</ResponseField>
    <ResponseField name="reward_type" type="string">Reward type enum value.</ResponseField>
    <ResponseField name="reward_percentage" type="number">Percentage value, or `null`.</ResponseField>
    <ResponseField name="reward_amount" type="number">Fixed amount, or `null`.</ResponseField>
    <ResponseField name="min_qty_tickets" type="number">Minimum ticket quantity threshold, or `null`.</ResponseField>
    <ResponseField name="min_amount_tickets" type="number">Minimum order amount threshold, or `null`.</ResponseField>
    <ResponseField name="locality_id" type="number">Scoped locality ID, or `null`.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Validation error or business rule violation. |
| `401`  | Missing or invalid Bearer token. |
| `403`  | `ORGANIZER` does not own the event. |

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.paypac.com.co/api/events/42/reward-rules \
  --header 'Authorization: Bearer <firebase-id-token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "reward_type": "PERCENTAGE",
    "reward_percentage": 10,
    "min_qty_tickets": 1
  }'
```

</CodeGroup>

## **POST** /api/reward-rules/calculate

<ParamField body="quantity" type="number" required>

</ParamField>

<ParamField body="total_amount" type="number" required>
  Total order amount (in minor currency units, e.g. cents).
</ParamField>

  Locality ID, to also check locality-scoped rules.
</ParamField>

<ResponseField name="hasReward" type="boolean">
  `true` if a matching rule was found, `false` if no applicable rules exist.
</ResponseField>

<ResponseField name="reward_type" type="string">
  The matched rule's reward type. Present only when `hasReward` is `true`.
</ResponseField>

<ResponseField name="reward_amount" type="number">
  Calculated reward amount in minor units. Present only when `hasReward` is `true`.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable explanation. Present when `hasReward` is `false`.
</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Validation error. |
| `401`  | Missing or invalid Bearer token. |

<CodeGroup>

```bash cURL

  --header 'Authorization: Bearer <firebase-id-token>' \
```
