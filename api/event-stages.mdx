---
title: "Event Stages"
description: "Manage pricing stages (ticket tiers) for event localities. Stages define time-bounded pricing periods — early-bird, regular, and last-minute — for a locality's ticket sales."
---

<Info>
  Stages belong to a [locality](/api/event-localities). Each stage has a name, a date range, and a price. Only one stage is active at any given moment (the one whose `date_start`–`date_end` window includes the current time). The active stage determines the ticket price shown to customers.
</Info>

All endpoints require a valid Firebase ID token: `Authorization: Bearer <token>`.

---

## **POST** /api/localities/:localityId/stages

Creates a new pricing stage for a locality. Stage date ranges must not overlap with existing stages on the same locality. `date_end` must be after `date_start`.

**Required role:** `ORGANIZER` (event owner) or `PAYPAC`

### Request headers

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

### Path parameters

<ParamField path="localityId" type="number" required>
  ID of the locality to add the stage to.
</ParamField>

### Request body

<ParamField body="stage_name" type="string" required>
  Display name for the pricing tier (e.g., `"Early Bird"`, `"Regular"`, `"Last Minute"`). Minimum 2 characters.
</ParamField>

<ParamField body="date_start" type="string" required>
  Start of the pricing window in ISO 8601 format (e.g., `"2026-06-01T00:00:00.000Z"`). Tickets in this locality are sold at `price_ticket` from this moment.
</ParamField>

<ParamField body="date_end" type="string" required>
  End of the pricing window in ISO 8601 format. Must be strictly after `date_start`.
</ParamField>

<ParamField body="price_ticket" type="number" required>
  Ticket price in the event's currency (integer, smallest unit — e.g., COP pesos). Must be a positive integer.
</ParamField>

### Response `201`

<ResponseField name="id" type="number">
  Auto-incremented stage ID.
</ResponseField>

<ResponseField name="locality_id" type="number">
  ID of the parent locality.
</ResponseField>

<ResponseField name="stage_name" type="string">
  Stage display name.
</ResponseField>

<ResponseField name="date_start" type="string">
  Start of pricing window, ISO 8601.
</ResponseField>

<ResponseField name="date_end" type="string">
  End of pricing window, ISO 8601.
</ResponseField>

<ResponseField name="price_ticket" type="number">
  Ticket price.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 last-update timestamp.
</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Validation error — e.g., `date_end` is before `date_start`, price is not a positive integer. |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Caller is not the event owner or does not have `PAYPAC` role. |
| `404`  | Locality not found. |

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.paypac.com.co/api/localities/7/stages \
  --header 'Authorization: Bearer <firebase-id-token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "stage_name": "Early Bird",
    "date_start": "2026-06-01T00:00:00.000Z",
    "date_end": "2026-07-01T23:59:59.000Z",
    "price_ticket": 80000
  }'
```

</CodeGroup>

  Firebase ID token: `Bearer <token>`
</ParamField>

### Path parameters

<ParamField path="localityId" type="number" required>
  ID of the locality.
</ParamField>

### Response `200`

Returns a single stage object, or `null` if no stage is currently active.

<ResponseField name="stage_name" type="string">Stage display name.</ResponseField>
<ResponseField name="date_start" type="string">Start of active window.</ResponseField>
<ResponseField name="date_end" type="string">End of active window.</ResponseField>
<ResponseField name="price_ticket" type="number">Current ticket price.</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `401`  | Missing or invalid Bearer token. |
| `404`  | Locality not found. |

<CodeGroup>

```bash cURL

  --url https://api.paypac.com.co/api/localities/7/stages/active \
  --header 'Authorization: Bearer <firebase-id-token>'
```

## **PUT** /api/stages/:id

Updates one or more fields of an existing pricing stage. If both `date_start` and `date_end` are provided, the updated range must still be valid (`date_end` after `date_start`).

**Required role:** `ORGANIZER` (event owner) or `PAYPAC`

### Request headers

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

### Path parameters

<ParamField path="id" type="number" required>
  Numeric stage ID.
</ParamField>

### Request body

All fields are optional. Provide only the fields to update.

<ParamField body="stage_name" type="string">
  Updated stage name. Minimum 2 characters.
</ParamField>

<ParamField body="date_start" type="string">
  Updated start datetime, ISO 8601.
</ParamField>

<ParamField body="date_end" type="string">
  Updated end datetime, ISO 8601. Must be after `date_start` if both are provided.
</ParamField>

<ParamField body="price_ticket" type="number">
  Updated ticket price. Positive integer.
</ParamField>

### Response `200`

Returns the updated stage object.

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Validation error (e.g., invalid date order). |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Caller is not the event owner or does not have `PAYPAC` role. |
| `404`  | Stage not found. |

<CodeGroup>

```bash cURL
curl --request PUT \
  --url https://api.paypac.com.co/api/stages/12 \
  --header 'Authorization: Bearer <firebase-id-token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "stage_name": "Super Early Bird",
    "price_ticket": 65000,
    "date_end": "2026-06-15T23:59:59.000Z"
  }'
```

</CodeGroup>

## **DELETE** /api/stages/:id

Permanently deletes a pricing stage.

**Required role:** `ORGANIZER` (event owner) or `PAYPAC`

<Warning>Deleting an active stage will remove the current ticket price for the locality. Ensure another stage covers the remaining sales window before deleting.</Warning>

### Request headers

<ParamField header="Authorization" type="string" required>
  Firebase ID token: `Bearer <token>`
</ParamField>

### Path parameters

<ParamField path="id" type="number" required>
  Numeric stage ID to delete.
</ParamField>

### Response `200`

<ResponseField name="message" type="string">
  Confirmation message.
</ResponseField>

### Error responses

| Status | Description |
|--------|-------------|
| `400`  | Cannot delete (e.g., stage has associated sold tickets). |
| `401`  | Missing or invalid Bearer token. |
| `403`  | Caller is not the event owner or does not have `PAYPAC` role. |
| `404`  | Stage not found. |

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url https://api.paypac.com.co/api/stages/12 \
  --header 'Authorization: Bearer <firebase-id-token>'
```

</CodeGroup>
